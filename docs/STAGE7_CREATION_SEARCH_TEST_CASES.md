# 새 회의 생성 및 단계 7 검색·필터 테스트 케이스

## 1. 구현 범위

- `/meetings/new` 새 회의 생성 화면
- 프로젝트, 회의 제목·유형, 참석자, 안건, 녹음 동의 입력
- 참석자와 안건 행 추가·삭제
- 생성 성공 후 방금 만든 회의의 녹음 작업대로 이동
- 회의 제목, 참석자, 최종 전사와 최종 회의록 통합 검색
- 프로젝트, 회의 상태·유형, 전사·회의록 확정 여부, 작성자, 기간 필터
- 검색 조건 URL 동기화, 결과 수, 빈 결과, cursor 페이지 이동

## 2. 실제 화면 테스트 케이스

| ID | 시나리오 | 실행 절차 | 기대 결과 |
|---|---|---|---|
| TC7-01 | 새 회의 진입 | 상단 `새 회의` 또는 목록의 `새 회의 만들기`를 누른다. | `/meetings/new` 화면과 활성 프로젝트 목록이 표시된다. |
| TC7-02 | 필수 입력 안내 | 제목, 참석자, 안건 또는 동의를 비우고 제출한다. | 공통 입력 안내에서 문제가 있는 항목을 구체적으로 알려주고 해당 입력칸으로 이동한다. |
| TC7-03 | 참석자·안건 추가 | 참석자와 안건 추가 버튼을 눌러 두 개 이상 작성한다. | 각 행을 독립적으로 작성하거나 삭제할 수 있다. |
| TC7-04 | 회의 생성 성공 | 기본정보와 동의를 입력하고 `회의 만들고 녹음 시작`을 누른다. | DB에 회의·참석자·안건이 저장되고 새 meeting ID의 작업대와 성공 안내가 표시된다. |
| TC7-05 | 권한·프로젝트 오류 | 권한 없는 계정 또는 사용할 수 없는 프로젝트로 생성한다. | 권한 요청 또는 프로젝트 재선택 방법을 설명하는 안내가 표시된다. |
| TC7-06 | 제목 검색 | 회의 제목 일부를 검색한다. | 일치하는 회의와 결과 수가 표시된다. |
| TC7-07 | 참석자 검색 | 참석자 이름을 검색한다. | 해당 참석자가 포함된 회의만 표시된다. |
| TC7-08 | 최종 전사 검색 | 확정 전사의 문장 일부를 검색한다. | 전사 내용이 일치하는 회의가 표시된다. 미확정 로컬 전사는 검색되지 않는다. |
| TC7-09 | 회의록 검색 | 회의록 제목·요약·핵심 내용·논의·결정·할 일 일부를 검색한다. | 해당 최종 회의록의 회의가 표시된다. |
| TC7-10 | 복합 필터 | 프로젝트, 유형, 전사 확정, 회의록 확정과 기간을 함께 지정한다. | 모든 조건을 만족하는 회의만 표시되고 조건이 URL query에 유지된다. |
| TC7-11 | 필터 cursor | 검색 결과가 10개를 넘을 때 다음 페이지로 이동한다. | 검색 조건이 유지되고 이전 페이지와 중복되지 않는다. |
| TC7-12 | 빈 검색 결과 | 일치하지 않는 검색어를 입력한다. | `검색 조건에 맞는 회의가 없습니다`와 초기화 버튼이 표시된다. |
| TC7-13 | 잘못된 검색 조건 | 100자 초과 검색어나 시작일보다 빠른 종료일 URL로 접속한다. | 서버 오류 대신 검색 조건을 확인하라는 안내가 표시된다. |
| TC7-14 | 조직 격리 | 다른 조직의 제목·전사·회의록 내용으로 검색한다. | 결과 수가 0이며 데이터 존재 여부가 노출되지 않는다. |
| TC7-15 | 반응형·접근성 | 모바일·태블릿·데스크톱에서 생성과 검색을 수행한다. | 입력 행과 필터가 한 열로 재배치되고 접근성 이름으로 모든 입력을 조작할 수 있다. |

## 4. 검색 실행계획 확인

로컬 데이터에서 `pg_trgm` 검색 인덱스를 사용할 수 있는지 다음 대표 조건으로 `EXPLAIN (ANALYZE, BUFFERS)`를 확인했다.

```sql
SELECT id FROM meetings WHERE title ILIKE '%회의록%';
```

확인 결과 `Bitmap Index Scan on meetings_title_trgm_idx`가 사용되었고 실행 시간은 로컬 기준 약 0.25ms였다. 실제 운영 데이터에서는 조직 조건, 검색어 분포와 통계에 따라 PostgreSQL이 더 저렴한 실행계획을 선택한다.
