# MeetingLoop AI 데이터 저장 및 개인정보 처리 정책

정책 버전: `2026-07-16`
구현 기준 migration: `0007_privacy_retention_operations.sql`

## 1. 적용 원칙

이 문서는 MeetingLoop AI의 화면 문구, API, 데이터베이스 및 로그가 따라야 하는 기준 문서다. 과거 문서나 구현 기록과 충돌하면 이 문서를 우선한다.

## 2. 서버 저장 범위

서버 DB에 저장하는 데이터:

- 사용자, 조직, membership 및 프로젝트
- 회의 제목, 일시, 상태, 참석자와 안건
- 사용자가 수정하고 `최종 전사 확정`을 실행한 전사 문장
- 최종 전사의 화자, 순서, 시작·종료 시각
- 사용자가 `회의록 최종 확정`을 실행한 회의록
- 전사와 회의록의 버전, 수정자, 확정자 및 수정 이력
- 녹음 동의 시각·동의 actor·정책 버전
- Gemini 전송 동의 시각·actor·회의·provider·정책 버전
- 회의 삭제 요청, 보존 만료 처리 및 영구 삭제 감사 이벤트

서버에 저장하지 않는 데이터:

- 원본 음성 파일 및 녹음 청크
- 브라우저에서 작성 중인 임시 전사
- 음성 인식 직후의 수정 전 전사 원문
- 최종 확정 전 AI 회의록 초안

원본 음성과 임시 전사는 브라우저 파일 또는 IndexedDB에만 존재한다. 애플리케이션은 원본 음성을 받는 서버 API와 재생 URL API를 제공하지 않는다.

`recordings` 테이블은 `LOCAL_ONLY` 메타데이터만 가지며 파일 경로, object storage key, checksum 또는 업로드 상태를 저장하지 않는다. 음질/VAD/overlap/화자 후보/사전/review 초안도 서버 테이블에 저장하지 않는다.

## 3. 상태 전이

### 전사

| 현재 상태 | 동작 | 다음 상태 | 서버 저장 |
|---|---|---|---:|
| `LOCAL_DRAFT` | 사용자가 문장 수정 | `LOCAL_DRAFT` | 아니요 |
| `LOCAL_DRAFT` | 최종 전사 확정 | `CONFIRMED` | 예 |
| `CONFIRMED` | 권한 사용자가 재수정 후 확정 | `CONFIRMED` 새 버전 | 예 |
| `CONFIRMED` | 삭제 요청 | `DELETION_PENDING` | 목록·API에서 즉시 숨김, 복구기간 후 영구 삭제 |

서버 API는 `CONFIRMED` 전사만 입력받는다. 수정 전 `rawText`는 요청과 DB 스키마에서 허용하지 않는다.

### 회의록

| 현재 상태 | 동작 | 다음 상태 | 서버 저장 |
|---|---|---|---:|
| `DRAFT` | AI 생성 및 사용자 편집 | `DRAFT` | 아니요 |
| `DRAFT` | 회의록 최종 확정 | `CONFIRMED` | 예 |
| `CONFIRMED` | 권한 사용자가 재수정 후 확정 | `CONFIRMED` 새 버전 | 예 |
| `CONFIRMED` | 삭제 요청 | `DELETION_PENDING` | 목록·API에서 즉시 숨김, 복구기간 후 영구 삭제 |

AI가 생성한 초안은 응답과 브라우저 상태에만 존재하며 최종 확정 전에는 영속 저장하지 않는다.

## 4. 역할별 권한

조직과 프로젝트 접근 범위 검증을 통과한 뒤 다음 권한을 적용한다.

| 역할 | 조회 | 최종 전사 확정·재수정 | 회의록 생성·재수정·확정 | 삭제 |
|---|---:|---:|---:|---:|
| `ORG_ADMIN` | 가능 | 가능 | 가능 | 가능 |
| `PROJECT_ADMIN` | 가능 | 가능 | 가능 | 현재 membership이 허용하는 조직 프로젝트 범위에서 가능 |
| `EDITOR` | 가능 | 가능 | 가능 | 불가 |
| `MEMBER` | 가능 | 불가 | 불가 | 불가 |
| `VIEWER` | 가능 | 불가 | 불가 | 불가 |
| `EXTERNAL` | 지정 회의만 가능 | 불가 | 불가 | 불가 |

화면의 버튼 노출과 관계없이 API와 repository에서 동일한 권한을 강제한다.

## 5. 보존 및 삭제

- 조직의 기본 보존기간은 회의 종료일 기준 365일이다.
- 조직 관리자는 향후 관리 화면에서 30~3650일 범위로 설정할 수 있다.
- 종료일이 없으면 회의 생성일을 보존기간 기준으로 사용한다.
- 사용자 삭제 요청은 30일 복구기간 동안 `DELETED` 상태로 보관한 뒤 영구 삭제한다.
- 삭제 요청은 `meeting_deletion_requests`에 actor, 요청 시각, 사유와 `purge_after`를 기록하고 회의를 즉시 일반 조회에서 숨긴다.
- 조직 보존기간이 먼저 만료되면 복구기간과 관계없이 정책 만료 작업 대상으로 처리한다.
- 영구 삭제 시 최종 전사, 회의록, revision 및 관련 검색 데이터도 함께 삭제한다.
- 원본 음성은 서버에 없으므로 사용자가 브라우저에서 직접 삭제한다.
- 운영 백업은 암호화하며 최대 35일 후 만료한다. 영구 삭제된 데이터는 백업 만료 후 복구할 수 없다.
- 법적 보존 요청은 별도 기능이 구현되기 전까지 지원하지 않는다.

자동 삭제는 worker의 `retention.purge` maintenance sweep이 담당한다. 최초 배포는 `RETENTION_SWEEP_ENABLED=false`로 dry-run과 backup을 확인한 뒤 명시적으로 활성화한다. 삭제 감사 이벤트에는 회의 제목·전사·회의록 본문을 포함하지 않는다.

## 6. AI 제공자 전송

- Ollama 로컬 모드에서는 최종 전사가 설정된 로컬 Ollama 서버로 전달된다.
- Gemini 모드에서는 최종 전사가 Google Gemini API로 전송된다.
- 원본 음성은 어떤 AI 제공자에도 전송하지 않는다.
- Gemini 선택 화면에는 `분석할 최종 전사 TXT가 Google Gemini API로 전송됩니다. 원본 음성은 전송되지 않습니다.`라고 표시한다.
- 사용자는 Gemini 실행 직전에 별도 checkbox로 동의해야 한다. 서버와 worker는 현재 정책 버전의 meeting/user consent가 없으면 `EXTERNAL_AI_CONSENT_REQUIRED`로 거부한다.
- 동의 기록은 전송 허가 기록이며 provider 성공을 의미하지 않는다. 생성 초안은 사용자가 최종 확정하기 전 PostgreSQL에 저장하지 않는다.

## 7. 로그 정책

로그에 기록할 수 있는 항목:

- 요청 ID, 조직 ID, 사용자 ID, 회의 ID
- API 경로, HTTP 상태, 처리시간
- AI 제공자 및 모델명
- 오류 코드와 재시도 횟수

로그에 기록할 수 없는 항목:

- 원본 음성 또는 Base64 데이터
- 전사 문장 및 전체 TXT
- 회의록 제목, 요약 및 본문
- AI 요청 prompt와 응답 전문
- 비밀번호, session token 및 API key

오류를 기록할 때는 고정 오류 코드와 필요한 식별자만 사용한다. request body, AI 요청 본문 및 DB row 전체 객체를 `console`이나 외부 오류 추적 도구에 전달하지 않는다.

## 8. 검증 기준

- 애플리케이션에 원본 음성 업로드 및 재생 URL API가 존재하지 않는다.
- 전사 저장 입력은 `editedText`와 `CONFIRMED` 상태만 허용한다.
- 수정 전 전사 `rawText`가 포함된 요청은 거부한다.
- 화면에 로컬 음성 보관, 최종 전사 확정 및 Gemini 외부 전송 범위를 안내한다.
- 녹음 동의와 Gemini 동의의 시각·actor·정책 버전을 감사 가능하게 저장한다.
- 회의 삭제 요청 후 일반 조회가 차단되고, 보존 worker 실행 후 연관 콘텐츠가 cascade 삭제된다.
- 테스트가 원본 음성 서버 경로의 부재와 최종 전사 입력 규칙을 검증한다.
