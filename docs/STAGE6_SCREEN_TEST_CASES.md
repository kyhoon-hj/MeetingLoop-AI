# 단계 6 회의록 목록·상세 화면 테스트 케이스

## 1. 목적과 범위

단계 6에서 구현한 회의록 목록 `/meetings`와 상세 `/meetings/[meetingId]` 화면을 실제 브라우저 기준으로 검증한다. 목록·상세 조회, 페이지 이동, 수정 진입, 오류 및 빈 상태, 조직 간 데이터 격리, 반응형 화면을 포함한다.

자동화된 화면 테스트는 `tests/e2e/meetings.spec.ts`에 있으며 Chromium 기반으로 모바일, 태블릿, 데스크톱 크기에서 실행한다.

## 2. 사전 조건

- PostgreSQL이 `localhost:5432/meeting`에서 실행 중이다.
- 루트 `.env`에 `DATABASE_URL=postgresql://postgres:pgpass@localhost:5432/meeting`이 설정되어 있다.
- migration이 적용되어 있다.
- 테스트 계정은 활성 조직 멤버이며 회의 조회 권한이 있다.
- 상세 조회 대상 회의에는 참석자, 안건, 최종 전사와 최종 회의록 v2 및 각 v1 수정 이력이 있다.

## 3. 실제 화면 테스트 케이스

| ID | 화면/상황 | 실행 절차 | 기대 결과 |
|---|---|---|---|
| TC6-01 | 목록 진입 | 로그인 후 상단의 `회의록 목록`을 누른다. | `/meetings`로 이동하고 접근 가능한 회의만 표시된다. |
| TC6-02 | 목록 정보 | 대상 회의 카드를 확인한다. | 회의 제목, 프로젝트명, 일시, 참석자, 전사·회의록 확정 상태와 버전, 최종 수정자·수정일이 표시된다. |
| TC6-03 | cursor 페이지 이동 | 11개 이상의 회의를 준비하고 `다음 회의 보기`를 누른다. | 다음 cursor 페이지가 열리며 첫 페이지와 중복된 회의가 없다. |
| TC6-04 | 상세 기본정보 | 목록에서 대상 회의를 누른다. | 정확한 회의 상세 URL로 이동하고 제목, 프로젝트, 일시, 참석자, 안건이 표시된다. |
| TC6-05 | 최종 전사 조회 | 상세 화면의 `최종 전사` 영역을 확인한다. | 확정 상태와 현재 버전 v2, 최종 수정된 발화자·전사 내용이 표시된다. |
| TC6-06 | 최종 회의록 조회 | 상세 화면의 `최종 회의록` 영역을 확인한다. | 확정 상태와 현재 버전 v2, 제목·요약·핵심 내용·논의 내용·결정 사항·할 일이 표시된다. |
| TC6-07 | 수정 이력 | 상세 하단의 수정 이력을 확인한다. | 전사와 회의록의 이전 버전 v1, 수정자와 수정 시각이 구분되어 표시된다. |
| TC6-08 | 편집 진입 | 상세 화면에서 `전사 및 회의록 편집`을 누른다. | 해당 회의 ID가 포함된 작업 화면으로 이동한다. |
| TC6-09 | AI 재생성 진입 | 상세 화면에서 `AI 회의록 다시 생성`을 누른다. | 해당 회의 ID의 회의록 작업 화면이 열리며 회의록 탭이 선택된다. AI 호출은 사용자가 생성 버튼을 눌렀을 때만 실행된다. |
| TC6-10 | 잘못된 cursor | `/meetings?cursor=invalid-cursor`로 접속한다. | 서버 오류 화면 대신 페이지 정보를 읽을 수 없다는 안내와 목록으로 돌아가기 동작이 표시된다. |
| TC6-11 | 없는/타 조직 회의 | 존재하지 않거나 다른 조직의 회의 ID로 상세 URL에 접속한다. | 데이터 존재 여부를 노출하지 않는 동일한 `회의를 찾을 수 없습니다` 안내가 표시된다. |
| TC6-12 | 빈 목록 | 조회 가능한 회의가 없는 조직으로 로그인해 목록에 접속한다. | 빈 화면 대신 `아직 조회할 회의가 없습니다` 안내와 회의 만들기 진입 동작이 표시된다. |
| TC6-13 | 반응형 화면 | 모바일, 태블릿, 데스크톱 크기에서 목록과 상세를 확인한다. | 카드와 상세 영역이 화면 너비에 맞게 재배치되고 목록 메뉴와 주요 버튼을 사용할 수 있다. |
| TC6-14 | 비로그인 접근 | 세션 없이 `/meetings` 또는 상세 URL에 접속한다. | 로그인 화면으로 이동하며 회의 데이터가 노출되지 않는다. |

## 4. 자동 검증 방법

```powershell
$env:DATABASE_URL='postgresql://postgres:pgpass@localhost:5432/meeting'
corepack pnpm db:migrate
corepack pnpm test:e2e
```

E2E 테스트는 실행마다 별도의 조직과 11개 회의를 DB에 생성하고, 로그인부터 목록·cursor·상세·수정 이력·편집 진입·오류 안내까지 실제 화면에서 확인한 뒤 테스트 데이터를 삭제한다.

## 5. 완료 판정

- 모든 viewport의 E2E 테스트가 통과한다.
- 다른 조직의 회의가 목록 및 상세에 노출되지 않는다.
- 잘못된 URL, cursor, 빈 데이터가 기술적인 오류 메시지 대신 사용자가 이해할 수 있는 안내로 표시된다.
- 목록에서 선택한 회의 ID가 상세 및 편집 화면까지 유지된다.
